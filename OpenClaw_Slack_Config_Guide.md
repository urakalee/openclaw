# OpenClaw Slack 配置指南 (最佳实践版)

本文档提供将 OpenClaw 接入 Slack 的**一次性配置流程**，避免反复安装和权限调整。

## 1. 创建 Slack App

1. 登录 [Slack API 后台](https://api.slack.com/apps)。
2. 点击 **"Create New App"** -> **"From scratch"**。
3. 输入 App 名称（例如 "Davinci"）并选择工作区。

## 2. 开启 Socket Mode (获取 App-Level Token)

*建议最先做这一步，这样配置事件订阅时就不需要提供 URL 了。*

1. 左侧点击 **"Socket Mode"**，打开 **"Enable Socket Mode"**。
2. 在弹窗中输入 Token 名称（如 "Socket Token"）。
3. 点击 **"Generate"**。
4. **复制 `xapp-` 开头的 Token**（这是 **App-Level Token**）。
5. 点击 "Done"。

## 3. 配置事件订阅 (Event Subscriptions)

*这一步会自动添加部分所需的读取权限。*

1. 左侧点击 **"Event Subscriptions"**，打开 **"Enable Events"**。
2. 展开 **"Subscribe to bot events"**，添加以下事件：
   - `message.im` (接收私信)
   - `app_mention` (接收 @消息)

*此时系统会自动提示添加对应的 User Scope，无需手动操作。*

3. 点击底部的 **"Save Changes"**。

## 4. 补充权限 & 开启聊天 (OAuth & App Home)

1. 左侧点击 **"OAuth & Permissions"**。
2. 滚动到 **"Bot Token Scopes"**，手动添加以下**写权限**（事件订阅不会自动加这些）：
   - `chat:write` (发送消息)
   - `files:write` (发送文件)
   - `im:write` (发起私信，可选但推荐)
3. 左侧点击 **"App Home"**。
4. 在 **"Show Tabs"** -> **"Messages Tab"** 下，**勾选 "Allow users to send Slash commands and messages..."**。
   *(这一步如果不做，用户无法给你发消息)*

## 5. 安装应用 (获取 Bot Token)

*所有配置完成后，再一次性安装。*

1. 回到 **"OAuth & Permissions"** 页面。
2. 滚动到顶部，点击 **"Install to Workspace"** 并授权。
3. **复制 `xoxb-` 开头的 Token**（这是 **Bot User OAuth Token**）。

## 6. 配置 OpenClaw

将两个 Token 填入配置：
- `botToken`: 第 5 步获取的 `xoxb-...`
- `appToken`: 第 2 步获取的 `xapp-...`
